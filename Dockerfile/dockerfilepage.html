3. Dockerfile

O que é Dockerfile?

Parabéns por chegar ao 3° assunto do GoDocker! Agora iremos explorar o assunto de Dockerfile.

Basicamente, os Dockerfiles, são scripts com uma série de comandos para criação de uma imagem . Neles, podemos  fazer uma série de coisas como executar comandos sh, criar variáveis de ambiente, copiar arquivos e pastas do host para dentro da imagem

Isso permite que você possa automatizar a criação de imagens para os contêineres da sua aplicação se basearem , uma vez que uma aplicação pode fazer o uso de vários Dockerfiles!

O Script

Um script de Dockerfile não tem extensão. É apenas o nome "Dockerfile". Isso já basta para o arquivo ser reconhecido como um script.

Vamos entender como podemos montar este script! O primeiro passo, é se ambientar com alguns comandos e seus respectivos significados.

Imagem: documentação oficial

Neste exemplo acima, temos um script de Dockerfile que está configurando uma imagem de node.js.

Então, explicando o exemplo acima:

  1. Primeiro, nós precisamos importar a imagem de base, juntamente com a sua versão. Para isso utilizamos o comando FROM com a sintaxe <imagem>:<versão>
  2. Agora, o diretório base da aplicação. Com o comando WORKDIR é possível criar uma pasta caso ela não existir, porém, caso exista, ela é usada do mesmo jeito.
  3. O comando COPY nos permite copiar um ou mais arquivos para outro local. Neste exemplo estamos copiando o contexto atual, representado por um “.” para o contexto do aplicativo.
  4. Utilizamos o RUN para rodar qualquer comando que faça sentido para o contexto da nossa imagem . O comando yarn install --production é utilizado no gerenciamento de pacotes do Yarn para instalar apenas as dependências de produção de um projeto Node.js, excluindo as dependências de desenvolvimento.
  5. O comando CMD no Dockerfile é utilizado para especificar o comando que deve ser rodado quando um contêiner iniciado a partir da imagem é executado. É a instrução que define o comando padrão a ser executado dentro do contêiner quando ele é iniciado. Recebe uma lista desses comandos como parâmetro.

O mais importante é compreender que, você pode usar o Dockerfile para criar sua imagem uma vez e usá-la como base para diversos contêineres. É como fazer um modelo de papel e usá-lo várias vezes.
